YouTube Data API v3 키 발급 및 설정 완벽 가이드 🎬

YouTube 자동 업로드, 왜 API 키가 필요할까요?

여러분이 만드신 소중한 영상 콘텐츠를 YouTube에 자동으로 업로드하고 관리하려면, 프로그램이 YouTube와 안전하게 통신할 수 있는 방법이 필요합니다. 바로 이때 사용되는 것이 YouTube Data API v3 키입니다. 이 API 키는 프로그램에게 "나는 권한을 가진 사용자야"라고 알려주는 일종의 비밀 열쇠와 같습니다.

이 가이드에서는 Google Cloud Platform(GCP)을 통해 YouTube Data API v3 사용 설정을 하고, 프로그램에 필요한 두 가지 중요한 정보, 즉 API 키와 OAuth 2.0 클라이언트 ID 및 보안 비밀 (JSON 파일)을 얻는 과정을 단계별로 상세히 안내해 드립니다. 조금 복잡해 보일 수 있지만, 차근차근 따라오시면 충분히 하실 수 있습니다!

중요 안내: YouTube Data API는 사용량에 따라 할당량(Quota) 제한이 있습니다. 개인적인 용도로 자동 업로드 프로그램을 사용하는 경우 일반적으로 충분하지만, 매우 많은 영상을 자주 업로드하는 경우 할당량 문제를 겪을 수 있습니다. 이 경우 Google Cloud Platform에서 할당량 상향 요청을 고려해야 할 수 있습니다.

준비물

  • Google 계정 (YouTube 채널이 연결된 계정)
  • 인터넷 연결이 가능한 PC

YouTube Data API v3 설정 및 키 발급 단계

1단계: Google Cloud Platform (GCP) 접속 및 프로젝트 선택/생성

먼저, 웹 브라우저를 열고 Google Cloud Platform 콘솔로 이동합니다.

Google Cloud Platform 바로가기

로그인 후, 화면 왼쪽 상단에서 프로젝트를 선택하거나 새 프로젝트를 생성합니다.

GCP 콘솔에서 프로젝트를 선택하거나 새 프로젝트를 만드는 화면입니다. 처음 사용하시는 경우, 새 프로젝트를 만들어주세요. (예: "내 유튜브 자동업로드")

2단계: YouTube Data API v3 사용 설정

프로젝트가 선택된 상태에서, 왼쪽 탐색 메뉴에서 'API 및 서비스' > '라이브러리'로 이동합니다.

API 라이브러리 검색창에 "YouTube Data API v3"를 입력하고 검색 결과에서 해당 API를 선택합니다. 그리고 '사용' 버튼을 클릭하여 API를 활성화합니다.

YouTube Data API v3를 검색하고 '사용' 버튼을 누르는 화면입니다.

3단계: API 키 생성

API를 활성화한 후, 프로그램이 API를 호출할 때 사용할 일반적인 API 키를 생성합니다. 왼쪽 탐색 메뉴에서 'API 및 서비스' > '사용자 인증 정보'로 이동합니다.

화면 상단의 '+ 사용자 인증 정보 만들기' 버튼을 클릭하고 드롭다운 메뉴에서 'API 키'를 선택합니다.

API 키가 생성되면 화면에 표시됩니다. 이 키를 복사하여 안전한 곳에 보관하세요.

4단계: OAuth 2.0 클라이언트 ID 생성 (가장 중요!)

YouTube에 동영상을 '업로드'하는 행위는 사용자 계정의 권한이 필요하므로, OAuth 2.0 인증 방식을 사용해야 합니다. 이를 위해 클라이언트 ID와 클라이언트 보안 비밀이 포함된 JSON 파일을 받아야 합니다.

다시 'API 및 서비스' > '사용자 인증 정보' 페이지로 이동하여 화면 상단의 '+ 사용자 인증 정보 만들기' 버튼을 클릭하고 'OAuth 클라이언트 ID'를 선택합니다.

4.1. OAuth 동의 화면 구성 (처음 설정 시)

만약 OAuth 클라이언트 ID를 처음 만드는 경우, 'OAuth 동의 화면'을 먼저 구성하라는 메시지가 나올 수 있습니다. 안내에 따라 다음 정보를 입력합니다:

  • User Type: '외부'를 선택합니다. (테스트 목적이므로)
  • 앱 이름: 프로그램 이름 (예: "내 유튜브 업로더")
  • 사용자 지원 이메일: 본인 이메일 주소 선택
  • 개발자 연락처 정보: 본인 이메일 주소 입력

나머지 필드는 대부분 기본값으로 두거나 비워두고 저장 후 계속합니다. '범위', '테스트 사용자' 설정은 지금 단계에서는 건너뛰어도 괜찮습니다.

4.2. OAuth 클라이언트 ID 유형 선택 및 생성

OAuth 동의 화면 구성 후 (또는 이미 구성된 경우), 클라이언트 ID 만들기 페이지로 돌아옵니다.

  • 애플리케이션 유형: '데스크톱 앱'을 선택합니다. (저희 프로그램은 데스크톱에서 실행됩니다)
  • 이름: 클라이언트 ID의 이름 (예: "내 유튜브 업로드 프로그램 클라이언트")

'만들기' 버튼을 클릭합니다.

4.3. 클라이언트 ID 및 보안 비밀 JSON 파일 다운로드

클라이언트 ID가 생성되면 'OAuth 클라이언트가 생성되었습니다'라는 팝업창이 뜨면서 클라이언트 ID와 클라이언트 보안 비밀번호가 표시됩니다. 여기서 가장 중요한 것은 JSON 파일을 다운로드하는 것입니다. 화면 우측 상단 또는 팝업창 내에 있는 JSON 다운로드 아이콘(보통 아래쪽 화살표 모양)을 클릭하여 client_secret_xxxx.json (또는 유사한 이름의) 파일을 다운로드합니다.

생성된 OAuth 클라이언트 정보와 JSON 파일 다운로드 버튼입니다.

매우 중요: 이 JSON 파일에는 민감한 정보가 포함되어 있으므로, 절대로 외부에 노출시키거나 버전 관리 시스템(예: Git)에 직접 커밋해서는 안 됩니다. 안전한 곳에 잘 보관하세요.

API 키 및 OAuth JSON 파일, 프로그램에 어떻게 적용하나요?

발급받은 정보들을 프로그램에 설정하는 방법은 다음과 같습니다:

  1. 프로그램 실행 및 환경설정 탭 이동: 유튜브 콘텐츠 자동 생성 프로그램을 실행하고, '환경설정' 탭으로 이동합니다.
  2. YouTube API Key 입력: 'YouTube API Key' 입력란에 위 3단계에서 생성하고 복사한 API 키를 정확하게 붙여넣습니다.
  3. YouTube OAuth JSON 파일 경로 지정: 'OAuth JSON 파일' 항목에서 'JSON 파일 찾기...' 버튼을 사용하여 위 4.3단계에서 다운로드한 client_secret_xxxx.json 파일을 선택하여 지정합니다.
  4. 설정 저장 및 인증/테스트: 모든 정보를 입력한 후, '모든 설정 저장' 버튼을 클릭하여 변경 사항을 저장합니다. 저장 후, '업로드 설정' 탭의 '계정 연동' 버튼을 클릭하여 YouTube 계정 인증 절차를 진행합니다.

마치며

여기까지 YouTube Data API v3 사용을 위한 키 발급 및 OAuth 클라이언트 설정 과정을 알아보았습니다. 다소 복잡한 과정이었지만, 이 설정을 통해 여러분의 콘텐츠를 더욱 효율적으로 관리하고 업로드할 수 있게 될 것입니다.
